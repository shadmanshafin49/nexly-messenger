Development Log
Nexly Texting App – Development Log
1. What the app is about
The Nexly Texting App is a full-stack mobile chat application built using React Native (Expo) for the frontend and a Node.js + Express + MongoDB backend.
It allows users to:
* Create accounts and log in.

* Search for other users.

* Open chat screens with them.

* Send and receive text messages (with planned support for images).
Essentially, this is an instant messaging app with real-time communication capabilities (socket.io planned).

________________


2. What we have done so far
Frontend (React Native / Expo Router)
   * login.js

      * Allows existing users to log in.

      * Contains a link to create a new account.

         * accountcreation.js

            * Lets users sign up with details.

            * On successful account creation → navigates to the dashboard.

               * dashboard.js

                  * Shows a search bar to find users by username.

                  * Clicking a search result navigates to the chat screen.

                     * chat.js

                        * Displays chat history between the logged-in user and the selected receiver.

                        * Shows receiver’s name at the top (with a back button).

                        * Allows sending new messages (text).

                        * UI has bubbles for “my messages” vs “their messages”.

                        * Input bar has text box + send button + placeholder photo button.

Backend (Node.js / Express / MongoDB)
                           * User routes (userRoutes.js)

                              * POST /api/users/register – create account.

                              * POST /api/users/login – login.

                              * GET /api/users/search?query=... – search users by username.

                                 * Message routes (messageRoutes.js)

                                    * GET /api/messages/:senderId/:receiverId – fetch chat history between two users.

                                    * POST /api/messages – send a new message.

                                       * MongoDB Models

                                          * User schema with username, password, etc.

                                          * Message schema with senderId, receiverId, message, createdAt.

________________


3. Issues we have faced so far
                                             1. Frontend navigation issues

                                                * At first, navigating from dashboard → chat was returning to the login screen.

                                                * Fixed by correctly using useLocalSearchParams and passing userId, receiverId, and receiverName during navigation.

                                                   2. Chat screen not loading

                                                      * chat.js originally showed “Loading chat…” and nothing else.

                                                      * Fixed by properly handling search params and fetching messages with useLocalSearchParams.

                                                         3. Undefined username / friend errors

                                                            * Early versions used searchParams incorrectly → caused Cannot read property 'username' of undefined.

                                                            * Resolved by switching to useLocalSearchParams.

                                                               4. Message sending issues

                                                                  * Messages were stuck in the input box, not appearing in chat.

                                                                  * Backend response shape mismatch caused Cannot read properties of undefined (reading 'split').

                                                                  * Fixed by normalizing the message object in chat.js and ensuring state updates.

                                                                     5. UI issues

                                                                        * Chat bubbles sometimes empty or not appearing at all.

                                                                        * Receiver’s name not visible on top bar.

                                                                        * Bubbles now display correctly, receiver name appears, but design still needs refinement.

                                                                           6. Image sending

                                                                              * UI button is ready (📷) but functionality not implemented yet.

                                                                              * Planned to integrate Expo ImagePicker with backend file upload.

________________
